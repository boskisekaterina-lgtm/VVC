# План перехода к обновлённой модели дивизиона ВВС

Документ описывает поэтапный переход от текущей модели работы к обновлённой бизнес-модели дивизиона «Вход в сети».

Основание: концепция, согласованная с собственником 1 июня 2025.

---

## Логика перехода

Переход строится по принципу «от аналитики к инструментам, от инструментов к пилоту, от пилота к системе».

```
Фаза 1 (нед. 1)     → Фаза 2 (нед. 2–3)    → Фаза 3 (нед. 4)     → Фаза 4 (нед. 5–6)   → Фаза 5 (пост.)
Аналитика и база     Инструменты и            Команда и             Пилот и               Масштабирование
поставщиков/сетей    регламенты               мотивация             корректировка         и развитие
```

---

## Фаза 1. Аналитика и инвентаризация базы
**Срок:** Неделя 1
**Ответственный:** Катя (координация) + вся команда

### Задачи

**1.1. Инвентаризация базы поставщиков**
- Выгрузить всех участников ЦЗС за последний год из платформы
- Проверить итоги переговоров: сколько встреч, какие сети проявили интерес, заключён ли контракт
- Проверить обороты и масштаб бизнеса по каждой компании

**1.2. Предварительный ABC-анализ поставщиков**
- Разделить на три категории:
  - **A** — много успешных переговоров на ЦЗС, но нет контракта; понятный продукт; есть логистика и документы
  - **B** — есть потенциал, но нужна доработка (упаковка, цена, объём, документы)
  - **C** — слабая готовность, не выводить в активный контур
- Составить список приоритетных поставщиков категории A для пилота (не менее 10–15 компаний)

**1.3. Инвентаризация сетей**
- Актуализировать реестр 74 сетей (64 food + 10 non-food)
- Зафиксировать по каждой: ответственный, уровень раппорта (слабый / рабочий / сильный / стратегический), последний контакт, категории интереса
- Выявить сети без ответственного или без актуального контакта

**1.4. Анализ текущей экономики**
- Собрать данные по действующим сделкам: модель оплаты (процент / разовый), участие Вартана, статус выплат
- Зафиксировать, какие сделки работают без Вартана

### Результат фазы
- Список поставщиков A/B/C с краткой характеристикой каждого
- Актуализированный реестр сетей с уровнями раппорта
- Список «горячих» поставщиков для пилота (категория A)

## Фаза 2. Разработка инструментов и регламентов
**Срок:** Недели 2–3
**Ответственный:** Лапшина / Подлеснов (контент) + Катя (контроль качества)

### Блок А. Инструменты для работы с поставщиками

**2.1. Дорожная карта поставщика**
Документ для клиента: что происходит на каждом этапе, сколько это стоит, какой результат считается выполненным, какие документы нужны, где возможны отклонения.

Структура:
- Этап 0: Диагностика (ABC-оценка, бесплатно или 50 000 руб.)
- Этап 1: Аудит КП и подготовка к коммуникации с сетью — 50 000 руб.
- Этап 2: Проверка предложения у выбранной сети (ответ по чек-листу) — 50 000 руб.
- Этап 3: Предварительная договорённость — 150 000 руб.
- Этап 4: Подписанный договор с сетью — 300 000–500 000 руб.
- Опция: Продолженное сопровождение на 1 год (% или смешанная модель)

**2.2. Форма ABC-анализа поставщика**
Таблица оценки по критериям:

| Критерий | Вес | A | B | C |
|----------|-----|---|---|---|
| Продукт и спрос | высокий | понятный, подтверждённый | есть, но нишевый | непонятный |
| Цена и маржинальность | высокий | проходит в сети | на грани | не проходит |
| Ассортимент | средний | широкий / глубокий | ограниченный | единичный |
| Логистика | высокий | есть своя / партнёр | частично | нет |
| Документы и сертификаты | высокий | полный комплект | частично | нет |
| Отсрочка | средний | готов 30–60 дн. | ограничен | не готов |
| Опыт с сетями / B2B | средний | есть | минимальный | нет |
| Объём производства | высокий | достаточный для сети | недостаточный | нет |

**2.3. Форма диагностики поставщика**
Краткое заключение после первичной оценки: можно выводить сразу / нужна доработка / не подходит сейчас. Фиксируется причина и рекомендуемый следующий шаг.

### Блок Б. Инструменты для работы с сетями

**2.4. Дорожная карта сети**
Описание логики взаимодействия с сетью:
1. Определение целевой сети и ЛПР
2. Проверка статуса раппорта
3. Подготовка КП в формате, удобном для сети
4. Проверка интереса по чек-листу
5. Встреча с поставщиком (при интересе)
6. Предварительная договорённость
7. Договор

**2.6. Форма ABC-анализа сети**
- **Категория A:** высокий раппорт, понятный ЛПР, готовность рассматривать через ВВС → персональное ведение, клубные мероприятия
- **Категория B:** рабочий контакт, ограниченный раппорт → поддерживать, подключать под конкретных поставщиков
- **Категория C:** нет контакта или высокий риск → не брать в активный контур

### Блок В. Модели работы

**2.8. Правила выбора модели монетизации**

| Условие | Дискретная воронка | Продолженная воронка |
|---------|-------------------|---------------------|
| Нет уверенности в платёжной дисциплине | ✓ | |
| Высокий потенциал оборота | | ✓ |
| Поставщик категории A с хорошей историей | | ✓ |
| Первое обращение, нет доверия | ✓ | |
| Поставщик вернулся после самостоятельной работы | ✓ повышенная | |
| Смешанная (по договорённости) | | ✓ |

**2.9. Модель работы с Вартаном**
- Вартан участвует: фиксируется до старта работы с поставщиком
- Вознаграждение: средний процент поставщикам 4%, делится 50/50 с Вартаном после оплаты
- Food / non-food разделяются в работе и экономике
- Если Вартан не участвует: ВВС ведёт самостоятельно по внутренней модели

**2.10. Модель работы без Вартана**
- Используется для non-food и части food-сетей
- ВВС выстраивает прямой раппорт через деловую полезность, клубные форматы, регулярную коммуникацию

### Результат фазы
- Дорожная карта поставщика (готова к показу клиенту)
- Дорожная карта сети (внутренний регламент)
- Чек-лист оценки КП (12 пунктов, готов к использованию с сетями)
- Формы ABC-анализа (поставщик и сеть)
- Форма диагностики поставщика
- Шаблон досье на сеть и ЛПР
- Правила выбора модели монетизации
- Описание моделей работы с Вартаном и без

## Фаза 3. Команда, мотивация и планирование
**Срок:** Неделя 4
**Ответственный:** Катя + руководитель дивизиона

### Задачи

**3.1. Закрепить роли и зоны ответственности**

| Сотрудник | Зона ответственности | Что считается результатом |
|-----------|---------------------|--------------------------|
| Вартан | Food-сети (основная часть) | Предварительная договорённость / договор по закреплённым сетям |
| Лапшина | Поставщики + часть сетей без Вартана | Диагностика, доведение до встречи |
| Подлеснов | Поставщики + co-ведение сетей | Диагностика, доведение до встречи |
| Катя | Контроль статусов, планирование, спорные ситуации | Актуальный статус по каждой сделке каждую неделю |
| Андрей | Ряд сетей (Метро, Командор, Перекрёсток, X5 и др.) | Ответ от сети по чек-листу / встреча |

**3.2. Разработать систему мотивации**

Мотивация привязана к объективно проверяемым результатам:

| Результат | Тип |
|-----------|-----|
| Привлечение поставщика категории A | Разовая премия |
| Проведение диагностики и выбор модели | Разовая премия |
| Получен ответ от сети по чек-листу | Разовая премия |
| Организована встреча поставщика и сети | Разовая премия |
| Зафиксирована предварительная договорённость | Разовая премия |
| Подписан договор | Повышенная премия |
| Удержание продолженной сделки (ежемесячно) | Регулярная премия |
| Развитие отношений с сетью категории A | Квартальная премия |

Оценка выполнения:
- Вовремя → положительная оценка и мотивация
- С опозданием → результат фиксируется, мотивация снижается
- Не выполнено по объективной причине → причина фиксируется, управленческое решение
- Не выполнено по субъективной причине → фиксируется ответственность, корректирующее действие

**3.3. Настроить недельный контур управления**

Формат еженедельного отчёта (ведёт Катя):

| Поставщик | Сеть | Этап | Ответственный | Срок | Стоимость этапа | Статус | Причина отклонения | Следующий шаг |
|-----------|------|------|---------------|------|----------------|--------|-------------------|---------------|
| ... | ... | ... | ... | ... | ... | ✓ / ✗ / → | объект. / субъект. | ... |

**3.4. Определить техническое решение**
Выбрать, где ведутся: статусы сделок, досье на сети и ЛПР, чек-листы, планы, финансовые этапы.
Варианты: Weeek, Notion, Битрикс24, Google Sheets + форматы.

### Результат фазы
- Зафиксированные роли и ответственность каждого сотрудника
- Таблица мотивации с размерами премий
- Шаблон еженедельного отчёта (заполненный первый раз)
- Выбранное техническое решение и настроенная структура

## Фаза 4. Пилот и корректировка
**Срок:** Недели 5–6
**Ответственный:** Лапшина / Подлеснов (исполнение) + Катя (контроль)

### Неделя 5. Запуск пилота

**4.1. Отобрать 5–10 поставщиков категории A из базы ЦЗС**
Критерии отбора:
- Провели 10+ переговоров на ЦЗС за последний год
- Сети проявляли интерес, но контракта нет
- Есть документы, логистика, стабильный объём

**4.2. Провести первые звонки**
Логика звонка:
1. Уточнить, удалось ли заключить контракт после ЦЗС
2. Выявить, на каком этапе застряли переговоры
3. Предложить конкретный этап сервиса ВВС (начиная с аудита КП — 50 000 руб.)

**4.3. Провести диагностику каждого поставщика**
Заполнить форму диагностики, присвоить категорию, выбрать модель монетизации.

**4.4. Запустить первые коммуникации с сетями**
По каждому поставщику категории A — обратиться в выбранную сеть, передать КП, запросить обратную связь по чек-листу.

**Цель недели 5:** заключить договорённости минимум с 3–5 поставщиками на любой из этапов дискретной воронки.

### Неделя 6. Разбор пилота и корректировка

**4.5. Провести разбор пилота**
Вопросы для разбора:
- Какие поставщики прошли воронку быстро, какие застряли и почему?
- Как сети реагировали на чек-лист? Что не работает в формате?
- Какие возражения по цене воронки? Что нужно скорректировать в тарифах?
- Что было сложно объяснить поставщику в дорожной карте?
- Как работает мотивация — что нужно скорректировать?

**4.6. Скорректировать инструменты**
- Чек-лист (если сети реагируют на отдельные пункты негативно)
- Тарифы (если цена входа слишком высока или занижена)
- Дорожная карта поставщика (если непонятна логика этапов)
- Систему мотивации (если критерии оценки неточны)

**4.7. Проверить гипотезу «5 клиентов в месяц»**
- Зафиксировать, сколько поставщиков зашли в воронку за 2 недели пилота
- Экстраполировать на месяц
- Если темп не достигается — определить причину (качество лидов / скрипт / цена / ресурс команды)

### Результат фазы
- Минимум 3–5 активных сделок в воронке
- Скорректированные инструменты (чек-лист, тарифы, дорожная карта)
- Выводы по пилоту для собственника (1 страница: гипотеза / факт / решение)

## Фаза 5. Масштабирование и развитие
**Срок:** После недели 6, постоянно
**Ответственный:** руководитель дивизиона + команда

### 5.1. Масштабирование воронки поставщиков
- Подключить дополнительные каналы лидогенерации: клубы, рассылки, дайджесты, сайт ВВС, чаты, выставки
- Запустить регулярную выгрузку участников ЦЗС + скрипт возврата поставщика
- Цель: стабильный поток — не менее 5 новых клиентов в месяц

### 5.2. Укрепление отношений с сетями категории A
- Сформировать программу касаний для ключевых ЛПР: деловые встречи, клубы, закрытые мероприятия, ужины, приглашения в Москву
- Обновлять досье на ЛПР после каждого контакта
- Цель: из рабочего контакта — в стратегический раппорт

### 5.3. Развитие самостоятельного направления (без Вартана)
- По каждой сети без закреплённого ответственного — назначить ответственного и запустить план касаний
- Развивать прямые отношения с non-food сетями

### 5.4. Клуб для ЛПР сетей
- Разработать концепцию закрытого клуба (деловые встречи, отраслевые ужины, экспертные сессии)
- Цель: сети воспринимают ВВС как полезного партнёра, а не поставщика поставщиков

### 5.5. Следующая ступень (долгосрочно)
- Самостоятельный сайт ВВС с описанием сервиса
- Дайджесты для сетей (что нового у поставщиков, интересные категории)
- Дайджесты для поставщиков (требования сетей, тренды, кейсы входа)
- Движение к международным сетям и контрактному производству

## Риски и меры

| Риск | Вероятность | Мера |

| Поставщики не готовы платить за дискретную воронку | средняя | Начинать с бесплатной или дешёвой диагностики, показывать ценность |
